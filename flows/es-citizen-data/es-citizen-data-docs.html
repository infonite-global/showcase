Overview

This technical guide provides integration patterns and reference documentation for the Spain Citizen Data extraction flow. It leverages the Infonite Widgets SDK to securely initialize sessions and handle user authentication via Cl@ve.

Simplified Documentation

This guide is a summary of the integration. For full specifications, error codes, and detailed schemas, please refer to the Official API Documentation.

Authentication

All API calls require a security token provided in the X-APP-SECRET header.

X-APP-SECRET: your_api_key_here

1. Session Initialization

Create a new session defining the features you want to extract and the configuration for the user experience.

POST /api/flows/es_citizen_data/v1/manager/init
API Reference

Request Payload

{
  "customer_id": "customer-123", /* Optional tracking ID */
  "settings": {
    "aes_key": "MTExMTExMTExMTExMTExMTEx...", /* Base64 (32 bytes) */
    "completion_mode": "redirect",
    "completion_url": "https://callback.com/success",
    "executions_features": [
      { "code": "customer_information_read" },
      { "code": "driver_data" },
      { "code": "labor_check" },
      { "code": "academic_data" },
      { "code": "public_pensions" },
      { "code": "yearly_tax_situation" }
    ]
  }
}
Domain Authorization Required

For security reasons, any callback domain provided in the completion_url parameter must be explicitly authorized. Please contact support@infonite.tech to whitelist your base hosts (e.g. https://www.example.com). Wildcards are not supported. Once the base host is authorized, you can dynamically append any path during session initialization (e.g. https://www.example.com/any/path).

AES-256-GCM Key Generation

You must provide a random 32-byte key (Base64 encoded) in settings.aes_key to ensure your users' data is encrypted securely for this specific session. Infonite stores this key only temporarily during the session lifespan.

Node.js require('crypto').randomBytes(32).toString('base64')
Python import os, base64; base64.b64encode(os.urandom(32)).decode()

2. Launching the Widget

Redirect the user to the URL provided (widget_url) in the initialization response. The user will be prompted to authenticate with Cl@ve and the extraction will take place interactively.

Iframe Embed & Events

If you embed the flow inside an HTML <iframe> instead of redirecting the top-level window, you can listen to Javascript postMessage events dispatched by the widget. This allows you to detect when the user finishes the process and gracefully close the iframe modal in your own frontend.

Cross-Origin Framing Policy

To protect against clickjacking, embedding the widget inside an iframe is blocked by default. You must request support@infonite.tech to whitelist your parent domains (e.g. *.example.com) so our engine can issue the correct Content-Security-Policy headers for your integration.

window.addEventListener('message', async (event) => {
    if (event.data) {
        // Detect workflow completion explicitly
        if (event.data.event === 'es_citizen_data:ended') {
            closeIframeModal();
            refreshDashboard();
        }
    }
});

3. Verification & Polling

Status polling is optional if you use Webhooks or URL callbacks, but it is useful if you want to reflect the session progression natively inside your own application's UI.

GET /api/flows/es_citizen_data/v1/manager/{session_id}
API Reference

This endpoint returns the current session status (e.g., STARTED, COMPLETED, FAILED) alongside detailed timestamps.

Session Cancellation

PATCH /api/flows/es_citizen_data/v1/manager/{session_id}/cancel
API Reference

You can force the cancellation of a session from your backend if needed (e.g. timeout on your side or user closing the tab before completion).

4. Webhooks & Decryption

Webhooks API

You can configure asynchronous server-to-server notifications to automatically receive the payload when the flow ends.

Available Events

  • es_citizen_data:started: Triggered when the user opens the widget footprint.
  • es_citizen_data:ended: Triggered when finished (success, error, or timeout) and contains the encrypted data results.

Webhook Payload

{
  "event_class": "EsCitizenDataV1FlowEvent",
  "producer": "es_citizen_data",
  "event": "es_citizen_data:ended",
  "session_id": "550e8400-...",
  "status_code": "COMPLETED",
  
  "data": "aes::U2FsdGVkX19..." // Encrypted Payload
}

Decryption (AES-256-GCM)

The string follows the syntax aes::Base64(IV + Ciphertext + Tag). To process it securely:

  1. Remove the prefix, then decode the Base64 portion.
  2. Extract the IV (first 12 bytes).
  3. Extract the Tag (last 16 bytes).
  4. Extract the Ciphertext (middle bytes).
  5. Decrypt it using your allocated Session AES key.

5. Synchronous Data Extraction

If you prefer not to use Webhooks, you can retrieve the plaintext result directly using your App Secret from the REST API once the session is marked as COMPLETED.

GET /api/flows/es_citizen_data/v1/manager/{session_id}/results
API Reference
